前言

设计模式(Design Pattern)是一套被反复使用、多数人知晓的、经过
分类编目的、代码设计经验的总结，使用设计模式是为了可重用代码、让代码更容易被他人
理解并且保证代码可靠性。

参考资料

  • 史上最全设计模式导学目录（完整版）, GitBook下载地址
  • 图解设计模式([日] 结城浩)
  • 模式和原则

设计模式有哪些

创建型设计模式 5个

  • Singleton(单例模式)
  • Factory Method(工厂方法模式)
  • Abstract Factory(抽象工厂模式)
  • Builder(建造者模式)
  • Prototype(模式)

结构型设计模式 7个

  • Adapter
  • Bridge
  • Composite
  • Decorator
  • Facade
  • Flyweight
  • Proxy

行为型设计模式 11个

  • Chain of Responsibility
  • Command
  • Interpreter
  • Iterator
  • Mediator
  • Memento
  • Observer
  • State
  • Strategy
  • Template Method
  • Visitor

题外话

16年上半年，还在实习那会，领导交给我一个任务，让我整理23种设计模式，当时也是参考了上面的博客，以及《Header First设计模式》
，不过仍然向看天书一样，足足看了大半个月，也只能算是一知半解。

随着工作经验和能力的积累，现在再回过头来看设计模式，发现原来平时的代码中就已经蕴含了许多设计模式，
比如我们的项目里面获取properties配置信息，就是使用的单例模式。

多学多看多练(^o^)/
