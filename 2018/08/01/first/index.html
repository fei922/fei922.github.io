前言

这两天心血来潮，想自己搭建一个blog，以便养成记录总结的习惯。以前认为没有记录的必要，找个笔记软件或者OneNote记录就好，
但是对于重度拖延症患者(如我)，笔记整理乱乱的，找起来也比较难，而且，知识这个东西，看明白和根据自己的理解正确表达出来是两码事。

既然有这么好的环境，那索性自己动手丰衣足食，哈哈。也是一个很好的督促。

网上教程一大堆了，就不再重复造轮子，贴一下大神教程，然后记录一下这其中遇到的一些问题及解决办法。

参考

  • Hexo中文文档
  • GitHub+Hexo 搭建个人网站详细教程
  • hexo优化及主题Landscape-F

搭建自己的blog

首先按照GitHub+Hexo 搭建个人网站详细教程的操作，先搭建起来了blog，
默认主题是landscape风格的。后来在了解添加标签的时候，
发现了landscape-f,由于这landscape-f是基于landscape主题扩展了一些使用功能，
比如标签、统计、目录、日历。所以决定切换到landscape-f主题。

可以结合这两篇参考文章，动手实践一下。这里说一下，我在实践中遇到的问题以及如何解决的吧。

遇到的问题以及解决方法

下载Hexo-cli时可以使用国内的npm源

我这里使用的淘宝的源

npm --registry https://registry.npm.taobao.org install -g hexo-cli

网络问题

执行这一步的时候hexo init blog，可能需要科学上网，刚开始执行的时候，一直卡在下载某个依赖包上，
后来使用代理之后就ok了，所以猜测可能是网络问题

之后的hexo命令执行操作都是在init指定的目录之下进行操作的，不然会提示hexo的用法不对（/捂脸。。）

比如我这里设定的”blog”，那么我要执行 hexo new my-first-blog，首先要cd hexo

将hexo与github关联起来

我这里按照GitHub+Hexo 搭建个人网站详细教程里面的设定，
在执行hexo d命令的时候，一直提示与github有关的Error: fatal: HttpRequestException encountered.发送请求时出错。
所以做了将https请求改成git协议

添加百度统计 (可选)

首先注册一个百度统计的账号，进入管理，设置监控你的blog

设置好后，会得到一段统计代码，需要放在blog页面的

标签里。这里统一放在”themes\landscape-f\layout_partial\header.ejs”里面

ps: 其实谷歌分析，也提供类似功能，不得不说，大厂的很多产品还是非常棒的。

注释landscape-f的日历组件

“blog\themes\landscape-f\layout_partial\after-footer.ejs” 这个文件在执行 hexo g的时候报错，
calendar组件定义里面存在”Cannot read property ‘single’ of undefined”，因此将该文件最下面和calendar有关部分删掉，
并且在”blog/_config.yml”中，将calendar功能注释。

后记

之前尝试过使用jekyll结合gitpage搭建个人博客，但是当时相关技术不太了解，就没有再继续下去，这一停就是一年多😂。
随着工作慢慢了解用了一段时间nodejs，这次使用Hexo搭建博客，发现也并没有那么难。其实，很多时候就是这样，只要认准了目标，
一步一步去做，原本模糊的概念会逐渐清晰，问题也总能解决的嘛。

要特别感谢howiefh大佬的主题，体验超级赞！

博客现在刚刚起步吧，各种摸索中，以后逐渐丰富~ ↖(^ω^)↗
